Minimum system requirements

The following hardware and software are required to use the Microsoft Windows Media Rights Manager 10 Software Development Kit (SDK):

  • Operating system

    Requirement: Microsoft Windows® Server 2003 with Internet Information Services (IIS)

  • Processor

    Requirement: 133-MHz or higher central processing unit (CPU), such as Intel Pentium or an Advanced Micro Devices (AMD) processor.

  • RAM

    Requirement: 256 megabytes (MB)

  • Free hard disk space

    Requirement: 4 MB

  • Video card

    Requirement: VGA or higher resolution

Installation notes

When you install the Windows Media Rights Manager SDK, files can be installed into the default installation directory, C:\WMSDK\WMRM10, or you can specify a different directory.

To install the Windows Media Rights Manager SDK
  • Run the installation program, Wmrm.exe.

When using Terminal Services in Application Server mode, you must install the Windows Media Rights Manager SDK through Control Panel so that the Help files are properly installed.

To install the Windows Media Rights Manager SDK through Control Panel
  1. Click Start, point to Settings, and then click Control Panel.
  2. Double-click Add/Remove Programs.
  3. Click Add New Programs.
  4. Navigate to Wmrm.exe to install Windows Media Rights Manager SDK.

For more information about how to use the Windows Media Rights Manager SDK, see the documentation (Wmrm_sdk.chm located in the \WMSDK\WMRM10 folder).

What's new in this release

The following list summarizes the new features and changes that were made in this release of the Windows Media Rights Manager 10 SDK:

  • Devices based on Windows Media DRM 10 for Portable Devices are supported. These devices can support secure clocks, so time-based licenses are enforced.
  • Metering allows content providers, such as subscription services, to accurately track content usage, thereby reducing royalty fees.
  • License chaining allows licenses to be divided into root and leaf licenses. This feature allows subscription services to quickly update a large number of licenses by reissuing a small number of root licenses rather than thousands of standard licenses.
  • License revocation allows services to delete specific licenses, which is useful for services that want to limit the number of computers on which a consumer can access their protected content.
  • New rights allow for new consumer scenarios, including sharing music in an online session and restricting how content is burned to CD according to playlists. Additionally, content owners have a greater degree of control over playback applications by requiring levels of security.

For more information, see What's New in the Windows Media Rights Manager 10 SDK documentation.
